مقدمة وحلقة ReAct

تم تطوير نظام ABAX لتقديم خدمات سفر شاملة تتضمن إدارة الحجوزات، التسعير، المخزون، التوصيات وغير ذلك. كل وكيل في المنصة يعمل ضمن دورة ReAct حيث يفكر، ثم ينفذ أداة معينة، ثم يلاحظ النتائج، قبل أن يقرر الخطوة التالية.

وكيل العمليات

يهتم هذا الوكيل بتأكيد الحجوزات، التنسيق مع الموردين، تعيين المرشدين، إدارة الشكاوى، ومراقبة الرحلات في الوقت الحقيقي.

  • تأكيد الحجوزات وإرسال القسائم
  • التواصل مع الموردين وتعيين المرشدين
  • تصنيف الشكاوى واقتراح حلول
  • مراقبة الرحلات وإرسال التنبيهات
[3] [4] [5]

تنفيذ وكيل العمليات ودورة الطلب

لتنفيذ وكيل العمليات في إطار Django مع نموذج OpenAI يمكن تعريف كل أداة كدالة Python مسجلة من خلال @method_tool. يستخدم الوكيل حلقة ReAct ليحدد الترتيب الصحيح للأدوات طبقاً للسياق. 

# تعريف أدوات وكيل العمليات (مختصرة)
@method_tool def verify_booking(booking_id: int) -> bool: ...
@method_tool def get_supplier_contract(supplier_id: int) -> dict: ...
@method_tool def generate_voucher(booking_id: int) -> str: ...
@method_tool def assign_guide(booking_id: int) -> dict: ...
# إنشاء وكيل ReAct وتنفيذ دورة تأكيد الحجز
ops_agent = create_react_agent(
    tools=[verify_booking, get_supplier_contract, generate_voucher, assign_guide],
    llm=openai_model
)
response = ops_agent("الرجاء تأكيد الحجز رقم 123 وتعيين مرشد")
print(response)

عند استقبال طلب تأكيد الحجز، يمر الوكيل بالمراحل التالية:

  • تحديد الحدث: استقبال حجز جديد أو تحديث الحجز، أو استلام شكوى.
  • جمع المعلومات: تحميل تفاصيل الحجز، مراجعة عقود الموردين، التحقق من السداد، وتحميل قائمة المرشدين المتاحين.
  • التخطيط: اختيار الأدوات المناسبة (مثل verify_booking، get_supplier_contract، assign_guide) ووضع تسلسل التنفيذ.
  • التنفيذ: استدعاء الأدوات واحدة تلو الأخرى؛ التحقق من الحجز ثم جلب العقد ثم توليد القسيمة وتعيين مرشد؛ التعامل مع الشكاوى بتصنيفها واقتراح الحلول.
  • التحقق: استخدام بوابة تأكيد لفحص نتائج كل أداة وضمان صحة البيانات؛ إذا حدث خطأ، يتم التصعيد للموظف البشري.
  • الإنهاء: تحديث الحجز في قاعدة البيانات، إرسال القسائم والإشعارات للعميل والمورد، وتوثيق العملية في السجلات.

تنفيذ وكيل المالية ودورة الطلب

يختص وكيل المالية بإصدار الفواتير، حساب العمولات، مطابقة الدفعات وكشف الاحتيال. يمكن تعريف هذه الوظائف كأدوات مستقلة ثم ربطها بوكيل ReAct.

@method_tool

    """يطابق دفعة البنك مع الفاتورة."""
    return True

@method_tool

# إنشاء الوكيل وتنفيذ الطلب المالي

تتبع دورة الطلب المالي المراحل التالية:

  • تحديد الحدث: إنشاء حجز جديد، إصدار فاتورة دورية، أو وصول دفعة من البنك.
  • جمع البيانات: استرجاع المعلومات المالية للحجز، أسعار العمولات، وضبط الحالة الحالية للفاتورة.
  • التخطيط: اختيار الوظائف الملائمة (generate_invoice، calc_commission، detect_fraud_signals، match_payment) وتحديد التسلسل.
  • التنفيذ: توليد الفاتورة، حساب عمولة الشركة، إرسال الفاتورة، ثم مطابقة الدفعات؛ تشغيل كاشف الاحتيال لمراقبة أي نشاط مشبوه.
  • التحقق: التحقق من تطابق المبلغ في الفاتورة مع المبلغ المسدد، ضمان عدم وجود إشارات احتيال؛ تحويل الحالات غير الطبيعية للمراجعة البشرية.
  • الإتمام: تحديث الحسابات في دفتر الأستاذ، إرسال إشعار للدفع للعملاء والموردين، وإعداد تقارير للأداء والتدفقات النقدية المستقبلية.

  • مقارنة الأسعار مع المنافسين

تنفيذ وكيل التسعير ودورة الطلب

يعتمد وكيل التسعير على مراقبة الطلب وسرعة الحجوزات وأسعار المنافسين لتوليد أسعار ديناميكية. تُعرّف الأدوات للحصول على المؤشرات وحساب السعر الأمثل ومراجعة الهامش ثم تنفيذ التحديث. 

# أدوات التسعير
@method_tool

@method_tool

    """يتحقق من عدم كسر هامش الربح الأدنى."""

تتبع دورة تحديث التسعير الخطوات التالية:

  • الحدث: انخفاض أو زيادة في سرعة الحجوزات أو تغير الأسعار لدى المنافسين.
  • جمع المؤشرات: باستخدام get_departure_metrics لجمع عدد الحجوزات والأيام المتبقية، و get_competitor_prices لجمع أسعار المنافسين.
  • التخطيط: تمرير المؤشرات لنموذج التسعير لتوليد السعر الأمثل عبر calculate_optimal_price، ثم تحديد هامش الربح الأدنى باستخدام check_margin_floor.
  • التنفيذ: إذا كان السعر آمن، تطبيقه بواسطة apply_price_update؛ وعند الحاجة، إنشاء حملة تسويقية أو تعديل خطة التسعير.
  • المراقبة: متابعة الأداء بعد التغيير، وتتبع أثر الأسعار على معدل الحجز؛ تشغيل نموذج التنبؤ لتحديث المعلمات باستمرار.
  • التحقق: الحصول على موافقة بشرية قبل أي تخفيض كبير لضمان الامتثال للسياسات.
  • التوثيق: تسجيل جميع التغييرات وإعلام الأقسام الأخرى (المخزون والمالية) عبر النظام الحدثي.

  • استقبال الطلبات وجمع السياق
  • التحقق من التوفر والأسعار والقواعد
  • حساب الغرامات والتكاليف
  • إنشاء/تعديل/إلغاء الحجز وإصدار الوثائق

تنفيذ وكيل الحجوزات ودورة الطلب

يتولى وكيل الحجوزات إدارة دورة حياة الحجز بالكامل، بما في ذلك إنشاء الحجز وتعديله أو إلغائه وفقاً للتوفر والسياسات. تُعرّف الأدوات للتحقق من التوفر وحساب التكلفة وتنفيذ الإجراءات. 

# أدوات الحجوزات
@method_tool
def check_availability(product_id: int, date: str) -> bool:
    """يتحقق من توفر مقعد أو غرفة في التاريخ المحدد."""
    return True

@method_tool
def calculate_booking_price(product_id: int, date: str) -> float:
    """يحسب السعر الإجمالي للحجز بما في ذلك الرسوم والضرائب."""
    return 1450.0

@method_tool
def create_booking(customer_id: int, product_id: int, date: str) -> dict:

# وكيل الحجوزات

يعمل الوكيل وفق دورة واضحة:

  • الحدث: طلب من العميل لإنشاء، تعديل، أو إلغاء حجز.
  • جمع السياق: استعمال check_availability للتأكد من التوفر في التاريخ المطلوب، وجمع السياسات والشروط المطبقة.
  • التخطيط: تحديد ما إذا كان الطلب إنشاء حجز جديد أو تعديل أو إلغاء، وحساب التكلفة أو الرسوم بواسطة calculate_booking_price أو calculate_modification_cost أو calculate_cancellation_penalty.
  • التنفيذ: استخدام create_booking لتأكيد الحجز، أو modify_booking لتعديل التاريخ، أو cancel_booking لإلغاء الحجز، ومعالجة المعاملات المالية المرتبطة.
  • التحقق: التأكد من تنفيذ العملية بنجاح، ومراجعة الأسعار والخصومات مع النظام المالي.
  • الإتمام: توليد الوثائق والفواتير عبر generate_booking_documents، إرسال الإشعارات للعملاء والموردين، وتحديث المخزون.
